# BountyHive Demo Fallback 预案（现场可执行应急手册）

> **版本**：v5 准确版对齐
> **原则**：主动诚实比被拆穿好；现场可控风险有预案，云端侧风险由 EvoMap 举办方处理。
> **适用场景**：3 分钟现场 Demo 期间出现的各类异常。

---

## 一、风险场景应急总表

| 风险场景 | 识别信号 | 应急动作 | 切换时间 | 负责人 |
|---|---|---|---|---|
| API 响应慢（>5s） | 讲解词卡顿、屏幕无响应 | 讲解人继续口播（不冷场）；后端操作员同步切预录视频 | <10s | 讲解人 + 后端操作员 |
| 真实模式超时 | 真实 EvoMap 调用 >10s 无响应或连续失败 | 切换 Mock 模式（`MOCK_MODE=true`）重跑，或切预录视频 | <15s | 后端操作员 |
| semantic-search 搜不到 failed Capsule | B 阶段（1:00）semantic-search 返回空 | 改用 `GET /a2a/assets?status=candidate` 列出 candidate 资产，手动定位 A 的 Capsule；口播"改用列表浏览" | <15s | 后端操作员 |
| 积分到账延迟 | 收尾阶段（2:35）`GET /billing/earnings/:agentId` 无新流水 | 展示预期流水截图 + 诚实标注"预期流水（fetch 奖励有延迟）" | <5s | 讲解人 |
| 能力链查询失败 | 高潮阶段（2:20）`GET /a2a/assets/chain/:chainId` 返回空或报错 | 展示本地维护的 chain_id 关系图（最后手段，需口播"这是本地缓存的能力链视图"） | <10s | 后端操作员 |
| Agent 节点掉线 | AgentStatusCard 灰点 | 心跳保活脚本自动重连；3 分钟 Demo 内不会掉线（文档说 15 分钟未活动才标记离线） | 自动 | 自动（心跳脚本） |

| 评委质疑"自己发布自己接" | Q&A 阶段提问 | 展示 3 账户登录截图 + `GET /a2a/directory` 页面截图，证明不同所有者 | <30s | 讲解人 |
| 现场网络断 | 任何阶段网络请求超时 | 切手机热点；若 10 秒内未恢复，切预录视频 | <30s | 后端操作员 |
| hello 限流（60 次/IP/小时） | hello 返回 429 | Demo 前已确认 3 节点 online，Demo 中不再 hello；若掉线重连被限流，切预录视频 | <5s | 后端操作员 |
| semantic-search 30 秒缓存延迟 | publish 后立即 search 搜不到 | Demo 脚本中 publish（0:35）到 search（1:00）已预留 25 秒 + 口播填充；若仍搜不到，用 candidate 列表兜底 | 自动 | 讲解人 |
| asset_id 哈希不匹配 | publish 返回 422 hash_mismatch | Demo 前已用 `POST /a2a/validate` 预检所有 Capsule；若现场仍报错，切预录视频 | <5s | 后端操作员 |
| owner 未启用"Agent 自主行为" | `/a2a/ask` 返回 403 | Demo 前 checklist 已确认启用；若现场报错，改用预录的 ask 响应截图 | <5s | 后端操作员 |

## 二、逐项详细预案

### 2.1 API 响应慢（>5s）

**识别信号**：
- 讲解人说完口播词，但屏幕动作未跟上
- 前端 Network 面板显示请求 pending 超过 5 秒
- 后端操作员终端无响应输出

**应急动作**：
1. 讲解人继续口播下一句（不要冷场，不要说"等一下"）
2. 后端操作员立即按快捷键切换到预录视频播放器（全屏）
3. 预录视频从当前时间点对应的位置开始播放（非从头播放）
4. 讲解人配合预录视频继续讲解，语气不变

**预防措施**：
- Demo 前 5 分钟用 `curl` 测一次 `GET /a2a/stats`，确认响应 <2s
- 后端预连接（keep-alive），减少 TLS 握手延迟
- 预录视频播放器始终打开在后台，快捷键已绑定

---

### 2.2 semantic-search 搜不到 failed Capsule

**识别信号**：
- B 阶段（1:00）`GET /a2a/assets/semantic-search?q=useEffect+dependency&outcome=failed` 返回空数组
- 前端搜索结果区域空白

**应急动作**：
1. 后端操作员立即调用 `GET /a2a/assets?status=candidate&type=Capsule&limit=20`
2. 在返回列表中手动定位 A 的 `capsule_lesson_burned_001`（按 signals_match 筛选 useEffect）
3. 前端切换到"candidate 列表"视图，高亮 A 的 Capsule
4. 讲解人口播："semantic-search 有 30 秒缓存延迟，我们改用 candidate 列表浏览——A 的失败 Capsule 就在这里，状态是 candidate，但可以被检索到。"

**预防措施**：
- Demo 前 30 分钟预发布一次 failed Capsule，等待 30 秒后用 semantic-search 确认能搜到
- 前端预先准备好"candidate 列表"视图切换按钮（隐藏快捷键）

---

### 2.3 积分到账延迟

**识别信号**：
- 收尾阶段（2:35）`GET /billing/earnings/node_agent_a` 返回的流水列表中没有 B fetch A 的新记录
- 前端收益面板显示"暂无新流水"

**应急动作**：
1. 讲解人不慌张，口播："fetch 奖励的到账有秒级延迟。这是预期流水——" 
2. 前端切换到"预期流水"视图（预先准备好的截图，标注"预期流水"水印）
3. 截图内容：B fetch A 的 Capsule → A 获 8 积分（GDI 61-80 分档）
4. 讲解人补充："实际的 fetch 调用是真实的，积分到账有延迟，我们诚实标注为预期流水。"

**预防措施**：
- Demo 前 30 分钟做一次跨账户 fetch 冒烟测试，确认积分能到账
- 预先准备"预期流水"截图，水印清晰
- 收尾阶段口播词预留"预期流水"的话术出口

---

### 2.4 能力链查询失败

**识别信号**：
- 高潮阶段（2:20）`GET /a2a/assets/chain/chain_react_useeffect_fix` 返回空或 404
- 前端能力链视图无数据

**应急动作**：
1. 后端操作员切换到本地维护的 chain_id 关系图（前端预备视图）
2. 关系图内容：A（capsule_lesson_burned_001, failed）→ B（capsule_success_001, success, reused_asset_id 指向 A）→ C（Capsule, success, parent 指向 B）
3. 讲解人口播："这是本地缓存的能力链视图。A 的失败 → B 的成功 → C 的复用，三节点共享 chain_id。"
4. **红线**：必须口播"本地缓存"，不能假装是实时查询结果

**预防措施**：
- Demo 前 30 分钟用 `GET /a2a/assets/chain/:chainId` 确认能查到完整链
- 前端预先用 publish 时的 chain_id 关系绘制本地关系图（基于真实 parent / reused_asset_id 字段）
- **仅作为最后手段**：此方案等于展示本地数据，评委查文档会发现非实时，必须诚实标注

---

### 2.5 Agent 节点掉线

**识别信号**：
- AgentStatusCard 从绿点变灰点
- 心跳脚本日志显示 timeout

**应急动作**：
1. 心跳保活脚本自动重连（无需人工干预）
2. 若 10 秒内未恢复，后端操作员手动重启心跳脚本
3. 讲解人继续口播，不提及掉线

**预防措施**：
- 心跳脚本设置为 60 秒间隔（文档默认 300 秒，Demo 期间缩短为 60 秒保活）
- 文档明确：15 分钟未活动才标记离线，3 分钟 Demo 内不会掉线
- Demo 前 5 分钟确认 3 节点全绿

---

### 2.6 bounty 流程某步骤卡住

**识别信号**：
- `POST /a2a/task/complete` 或 `POST /a2a/task/accept-submission` 返回非 200 或无响应
- 前端"任务完成"状态不更新

**应急动作**：
1. 跳过该步骤（complete / accept-submission），直接进入下一阶段
2. 后端操作员手动触发 publish 链路展示（B 的 success Capsule 已预发布）
3. 讲解人口播："Demo 中为加速展示，使用手动接受；生产环境走完整四维评估（AI 35% + Agent 投票 25% + 社区投票 15% + GDI 25%）。"
4. 能力链展示不受影响（chain_id 已在 publish 时建立）

**预防措施**：
- Demo 前 30 分钟跑一次完整 bounty 流程（ask → claim → publish → complete → accept-submission），确认端到端通畅
- 这是 v5 唯一命门：bounty 流程端到端。若 Demo 前未跑通，必须切预录视频
- B 的 success Capsule 预发布一份作为 fallback，确保能力链展示有数据

---

### 2.7 评委质疑"自己发布自己接"

**识别信号**：
- Q&A 阶段评委提问："你们的 A/B/C 是不是同一个账户？是不是左手倒右手？"

**应急动作**：
1. 讲解人调出 3 账户登录截图（3 个不同邮箱 + 3 个不同 node_id）
2. 调出 `GET /a2a/directory` 页面截图，展示 A/B/C 三个独立节点
3. 口播："A/B/C 分属三位不同队员，不同邮箱注册，不同 node_id。EvoMap 的环形交易检测会拦截同一所有者的自买自卖，我们的设计完全合规。"
4. 若评委追问，展示 `GET /a2a/nodes/:nodeId` 三个节点的声誉详情，证明独立身份

**预防措施**：
- 3 账户用不同邮箱注册（最好不同网络：A 用公司 WiFi、B 用手机热点、C 用另一台设备）
- Demo 前截图保存：3 账户登录页 + directory 页 + 3 节点声誉页
- 截图打印成 A4 纸，Q&A 时可直接递给评委

---

### 2.8 现场网络断

**识别信号**：
- 任何 API 请求超时（>10s 无响应）
- 前端 Network 面板全部 failed

**应急动作**：
1. 后端操作员立即切手机热点（预先绑定快捷键切换网络）
2. 若 10 秒内热点未连通，切预录视频
3. 讲解人口播："现场网络波动，我们切换到预录视频继续演示。"（诚实告知）

**预防措施**：
- 手机热点 Demo 前已开启，流量充足（至少 500MB）
- 预录视频已下载到本地，不依赖网络
- 后端操作员熟练掌握网络切换快捷键

## 三、预录视频清单（Fallback 用）

完整 3 分钟预录视频应包含以下 5 个高光时刻截图，作为切视频时的关键帧定位：

| 序号 | 时间点 | 高光时刻 | 截图内容 | 用途 |
|---|---|---|---|---|
| 1 | 0:35 | 失败独白 | A 的 failed Capsule publish 响应 + 失败独白卡片（暗红色背景，打字机文字） | 冲突阶段 fallback |
| 2 | 1:00 | 发现瞬间 | B 的 semantic-search 结果 + 失败经验星图中 A 的 Capsule 被点亮（金色高亮） | 进化阶段 fallback |
| 3 | 1:30 | 溯源声明 | B 的 success Capsule publish 响应，高亮 reused_asset_id / parent / chain_id 字段 | 进化阶段 fallback |
| 4 | 2:20 | 能力链生长 | `GET /a2a/assets/chain/:chainId` 返回的完整链 + A→B→C 三节点生长动画 | 高潮阶段 fallback |
| 5 | 2:45 | 蜂群热力图 | 蜂群进化热力图全屏，BountyHive 接过的任务节点点亮，热力图从红转绿 | 收尾阶段 fallback |

**预录视频技术要求**：
- 分辨率：1920×1080，60fps
- 时长：3 分 00 秒（与现场脚本严格对齐）
- 格式：MP4（H.264），本地可离线播放
- 播放器：VLC 或 PotPlayer，快捷键已绑定（空格暂停/播放，← → 快退/快进 10s）
- 备份：U 盘 + 本地硬盘双备份

## 四、Fallback 触发决策树

```
现场异常发生
├── 网络相关？
│   ├── 是 → 切手机热点（10s）
│   │   └── 10s 未恢复 → 切预录视频
│   └── 否 → 继续
├── API 响应慢（>5s）？
│   ├── 是 → 讲解人继续口播 + 切预录视频（10s）
│   └── 否 → 继续
├── 搜不到 / 查不到数据？
│   ├── semantic-search 空 → 切 candidate 列表（15s）
│   ├── 能力链空 → 切本地关系图 + 诚实标注（10s）
│   └── 积分无流水 → 切预期流水截图 + 诚实标注（5s）
├── bounty 流程卡住？
│   ├── 是 → 跳过该步骤 + 口播"加速展示"（20s）
│   └── 否 → 继续
└── 节点掉线？
    ├── 是 → 心跳自动重连（无需干预）
    └── 否 → 继续
```

## 五、诚实告知红线

以下情况必须主动诚实告知，不得掩饰：

| 情况 | 必须说的话 | 不能说的话 |
|---|---|---|
| 切预录视频 | "现场网络/API 波动，我们切换到预录视频继续演示。" | "这是实时画面。" |
| 展示预期流水 | "fetch 奖励有秒级延迟，这是预期流水。" | "这是真实到账的积分。"（若实际未到账） |
| 展示本地能力链 | "这是本地缓存的能力链视图。" | "这是实时查询结果。" |
| 跳过 bounty 步骤 | "Demo 中为加速展示，使用手动接受；生产环境走完整四维评估。" | "完整流程就是这样。" |
| GDI 分数展示 | "GDI 的 usage 维度基于 30 天滚动窗口，Demo 内展示的是预期影响。" | "GDI 从 32 升到了 61。"（动态数字） |

> **核心原则**：主动诚实比被拆穿好。评委查文档会发现真实情况，主动标注"预期/缓存/加速展示"反而加分。
